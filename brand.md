## Typography — JetBrains Mono

- **Primary (UI + body):** JetBrains Mono via `next/font/google` (`--font-jetbrains-mono`)
- **Fallback:** Arial, Helvetica, system mono

Technical / terminal feel. Prefer tabular numbers for ranks, counts, and wallet truncation.

| Role | Guidance |
|---|---|
| Display | 1.75–2.25rem, weight 700, tight tracking |
| Page title | 1.35–1.5rem, weight 700 |
| Section | 1rem, weight 600, uppercase optional with letter-spacing |
| Body | 0.85–0.9rem, line-height 1.6 |
| Caption | 0.7–0.75rem, color `--brand-muted` |
| Mono labels | 0.65–0.7rem, uppercase, letter-spacing 0.1–0.16em, primary cyan |

## Voice & tone

- **Builder-first, no fluff.** Short sentences. Prefer ranks, counts, and links over adjectives.
- **On-chain precise.** Use exact product names: SeekerID, `.skr`, Solana Mobile, dApp Store.
- **Unofficial but serious.** Never claim official Solana Mobile affiliation.
- **No hashtags** in product or marketing copy (house rule). Avoid emoji in UI unless the surface is explicitly chat/Telegram.

## Dos

- Use cyan (`#00ffd9`) for CTAs, focus, links, key metrics
- Keep dark teal/black bases for long data sessions
- Grid / phosphor accents sparingly (hero, empty states)
- Tabular nums for ranks and totals

## Don'ts

- Don't use purple/orange sol.new palette on this product
- Don't drop pure white body text on pure black without elevation hierarchy
- Don't use Comic / display novelty fonts
- Don't mark official Solana Mobile branding
